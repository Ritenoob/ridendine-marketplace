## Infrastructure changes (Vercel + Stripe — applied directly via API, no code change)

### ops-admin Vercel env vars
- **`CRON_SECRET`** — was set to empty string (silently broke every Vercel cron). Replaced with the real 64-char value from `apps/ops-admin/.env.local`. Production + Preview scopes.
- **`ENGINE_PROCESSOR_TOKEN`** — same fix, same scopes.
- Triggered a redeploy of ops-admin so the new values take effect.

### Stripe webhook configuration
- **Zero webhook endpoints existed in Stripe before this session.** Customer order → Stripe payment intent → `payment_intent.succeeded` event was generated by Stripe but no endpoint to receive it, so the engine's `payment_authorized → pending` auto-transition never fired and orders never reached chefs.
- Created webhook endpoint `we_1TX44mCDctkR5kafYpMfphCG` pointing to `https://ridendine.ca/api/webhooks/stripe`.
- Subscribed to: `payment_intent.succeeded`, `payment_intent.payment_failed`, `charge.refunded`, `transfer.created`, `payout.paid`, `payout.failed`.
- New signing secret installed as `STRIPE_WEBHOOK_SECRET` on the web Vercel project (replacing the stale value that didn't match any endpoint).
- Redeployed web.

### Data cleanup
- 3 abandoned draft orders + 1 stuck `payment_authorized` order from 2026-05-14 manually cancelled in DB. None had captured payments — the `payment_authorized` order's PaymentIntent was actually in Stripe's `requires_payment_method` state, confirming the customer never finished checkout.
